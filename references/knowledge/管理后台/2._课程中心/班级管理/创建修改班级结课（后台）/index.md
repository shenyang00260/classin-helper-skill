# 创建/修改/班级/结课（后台）


# 一、创建班级


### 适用人员


“后台管理系统”主账号


拥有“班级管理”权限子账号


### 使用说明


在“管理后台”中创建的班级，能够进行更多个性化设置。例如，可以添加插班生、调出生和旁听生等操作。此外，“管理后台”中还拥有更换班主任等功能。因此，建议用户在需要进行复杂管理操作时，优先选择在“管理后台”中创建和管理班级。


创建班级时，都有哪些参数可以设置，请参考下方


| 设置 | 说明 |
| --- | --- |
| 班级名称 | 必填项，为“班级”设置一个易于理解的名称，不得超过40个字符。 |

| 班级教师 | 班级中的教师，可以进行授课或辅助管理等，也可以建立完班级后，点击成员进行设置班级教师。 |
| 班级学生 | 班级中的学生，可以参与班级活动上课等，也可以建立完班级后，点击成员进行设置班级学生。 |

| 班级教室设置 | 为“班级”选择一个自定义教室配置，该配置将适用于该班级下所有“课堂”。如果未选择将默认使用学校设置的“默认自定义教室配置” |
| 班级标签 | 为“班级”添加标签，方便管理和搜索。 |
| 班级有效期 | 为“班级”设置一个有效期，到期后如果“班级”下没有未开始或正在进行中的“课堂”，系统会对该“班级”自动结课。

| 高级设置 | 可以开启或关闭“允许学生主动加入班级”、“允许班级成员互相添加好友”、“允许教师添加课节/课堂”、“允许学生修改班级昵称”、“离开班级的学生或班级解散后，可查看课程内容”、“允许班级成员开启临时教室”等功能。班主任在ClassIn软件中的班级中也可以进行设置。 |

# 三、班级结课


### 什么是“结课”？


“结课”的意思是指教务老师在ClassIn后台管理系统中对某个班级进行结课操作，ClassIn软件中对应的班级也将会同步解散。反之当ClassIn软件中的班级被解散后，系统会自动将后台管理系统中的对应课程进行结课。因此“结课”和“解散班级”是相同的意思，只是一个需要在后台管理系统中进行操作，另一个需要在ClassIn软件中进行操作。


### 适用场景


如果一个教务老师想要在ClassIn软件中解散一个班级，但是教务老师不是这个班级的班主任，那么教务老师就无法在ClassIn软件中进行操作。此时教务老师可以登录“管理后台系统”，找到对应的班级并将其结课。一旦班级结课，该班级所对应ClassIn软件中的班级也会被解散。


“管理后台”中的“结课”和软件中的“解散班级”指的是同一件事情。如果在“管理后台”中进行了“结课”操作，那么软件中对应的班级也会被“解散”。反之如果软件中“解散”了班级，在“管理后台”中对应的班级也会被“结课”。但是如果课程中还有未开始或未结束的课节时时，是无法进行“结课”操作的。当班级被结课后，班级内的学生和教师也不能再通过班级群进行交流。但是这并不影响他们观看班级中的录课回放视频。


结课后，如果您希望学生能够继续观看班级中的历史回放视频，请启用《允许离开班级的学生或班级解散后，可查看班级内容设置》功能。


班级无法被删除，只能被结课。
