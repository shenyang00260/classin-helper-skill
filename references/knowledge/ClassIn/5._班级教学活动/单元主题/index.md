# 单元主题


英文使用说明


# 一、什么是「单元主题」？


在课程设计中，我们常常以主题为中心整合教学内容，形成互相关联的系列教学活动，这些活动通常需要经过若干课时的完整呈现，我们称之为“单元”。需要注意的是，这里的“单元"并不完全等同于传统教科书中的“章节”， 而“主题”也不仅仅是教科书上的单元或章节标题。


# 二、新建「单元主题」


## 适用人员


“班主任”和“教师”


## 使用说明


一个单元主题可以包含多种教学/学习活动，例如：课堂教学、录播课、作业、测验(试卷)、答题卡、打卡、讨论、学习资料和AI口语卡等。通过这些多样化的活动，学生能够在一个连贯的主题框架内，全面理解和掌握相关知识。

# 六、「单元与活动」排序


所有人


### 适用场景


当班级中的“单元主题”或“学习活动”较多时，老师或学生可以切换排序查找“单元主题”或“学习活动”。


班级课程中的排序支持“正序排列”和“倒序排列”。


正序排列是按照主题或活动的创建时间，以远到近的方式排列。


倒序排列是按照主题或活动的创建时间，以近到远的方式排列。


可以参与排序的有：单元主题、课堂、录播课、作业、测验、答题卡、讨论、学习资料、打卡、AI口语卡等。

# 七、筛选活动


学习活动的筛选分为三种方式进行，以满足不同的需求和要求。以下是三种筛选方式：


条件筛选：可以根据特定的条件进行筛选，例如教师需要筛选与自己有关的所有教学活动，学生需要筛选某个特定教师名下的所有教学活动，或者筛选尚未开始、进行中或已结束的教学活动。此外，学生还可以通过此功能筛选自己待完成的学习活动，或根据“单元主题”以及“素养能力”等条件进行筛选。通过组合不同的条件进行筛选，可以更加精确地定位需要的学习活动。


类型筛选：可以根据活动的类型进行筛选，例如：课堂（课节、直播课）、录播课、作业、打卡、测验（试卷）、答题卡、讨论、学习资料、AI口语卡等。这种筛选方式能够帮助用户更快地找到特定类型的教学/学习活动。


关键词搜索：可以通过输入关键词来搜索教学/学习活动，以便快速找到相关的活动。


# 八、成绩


“班主任”、“教师”和“班级正式学生”


班级各课程均有成绩页：教师端含学习活动和能力模型，点正式学生会跳转至学生的课程报告中；学生点击成绩会跳转至自己的课程报告中，切换单元后可显示其单元报告；


教师用手机查看成绩页，仅显示学生总成绩和进度，点击成绩跳转至其课程报告，切换单元后可显示其单元报告；


学习活动：教师可查看所选课程下评分活动的学生总成绩及参与评价情况，支持按单元、活动类型等筛选；电脑端可导出当前页面内容，还能在评价设置中管理修改总成绩评分方案等；点击学生可跳转至对应学生的单元报告 - 学情分析；


能力模型：教师可查看所选课程下涉及评价量规的学生能力模型情况，支持搜索学生和能力模型；电脑端可导出当前页面内容，还能在评价设置中管理设置能力模型；点击学生可跳转至对应学生的单元报告 - 能力分析。

# 九、单元报告


单元报告是学生阶段性学习总结与评价，含学情分析、能力分析两大页面。


学情分析页包含单元活动总结（活动完成度、课堂出勤等数据）、能力分析、学习过程（单元所有活动完成情况，点击活动可跳转活动详情或活动报告）及教师评语；能力分析页按所选能力模型，直观展示能力图谱与列表。


学情分析和能力分析均支持分享：电脑 / 平板可分享至 ClassIn 班级 / 好友、复制链接，或微信扫码分享；手机可额外分享至朋友圈、微博、QQ。
